### **Aim**
> To study and implement various data constraints in SQL

### **Basic Syntax**
- **NOT NULL:** This constraint enforces a column to NOT accept NULL values
  ```sql
      column_name datatype NOT NULL
  ```
- **UNIQUE:** This constraint ensures that all values in a column are different
  ```sql
      column_name datatype UNIQUE
  ```
- **PRIMARY KEY:** This constraint uniquely identifies each record in a table
  ```sql
      column_name datatype PRIMARY KEY
  ```
- **FOREIGN KEY:** This constraint is used to link two tables together
  ```sql
      FOREIGN KEY (column_name) REFERENCES parent_table(parent_column)
  ```
- **CHECK:** This constraint ensures that all values in a column satisfy a specific condition
  ```sql
      column_name datatype CHECK (condition)
  ```
- **DEFAULT:** This constraint provides a default value for a column when none is specified
  ```sql
      column_name datatype DEFAULT default_value
  ```

### **Example Script**
```sql
CREATE TABLE Departments (
    DepartmentID INT PRIMARY KEY,
    DepartmentName VARCHAR(50) NOT NULL
);

CREATE TABLE Employees (
    EmployeeID INT PRIMARY KEY,
    FirstName VARCHAR(50) NOT NULL,
    LastName VARCHAR(50) NOT NULL,
    Email VARCHAR(100) UNIQUE,
    Age INT CHECK (Age >= 18),
    DepartmentID INT,
    Salary DECIMAL(10, 2) DEFAULT 30000.00,
    FOREIGN KEY (DepartmentID) REFERENCES Departments(DepartmentID)
);
DESCRIBE Employees;
```
