## **1. Introduction**
Modern household power management is hampered by devices that compromise security for convenience, or devices that offer robust safety but alienate everyday users with impenetrable data displays.  
**Project E.T.H.E.R.** is an innovative dual-node internet-of-things (IoT) ecosystem engineered to solve this dichotomy. By decoupling the high-voltage measurement hardware from the internet-facing user interface via an ESP-NOW wireless air-gap, **E.T.H.E.R.** prevents severe network vulnerabilities. At its core, localized edge computing instantly triggers power cutoffs for unsafe current loads without relying on cloud latency.  
Telemetry is securely funneled through serverless backend architecture into a high-performance, aesthetically pleasing React/Astro dashboard. Crucially, the system employs Large Language Models(API) to translate numerical data into relatable, "emotive" states visually represented by a sub-node, gamifying energy conservation for everyday consumers.

---

## **2. Problem Statement**
The proliferation of smart home appliances has exposed critical vulnerabilities in domestic energy monitoring and control. These flaws manifest in three primary areas:
  -	**Network Security Risks:** Consumer "smart plugs" typically connect high-voltage switching relays directly to domestic Wi-Fi, creating a perilous vector for cyber intrusions.
  -	**Delayed Emergency Response:** Traditional smart relays rely heavily on cloud-based processing for automation; in the event of an acute 220V power surge, this round-trip latency can result in catastrophic electrical failures.
  -	**Ineffective User Experience:** Existing platforms deliver sterile metrics and raw data that fail to engage non-technical households, leading to a profound lack of sustained energy conservation habits.
 
---

## **3. Literature Review**
### **3.1 Commercial Smart Connectors**
Devices such as TP-Link Kasa and Wemo utilize direct-to-router connections for remote power logging and actuation.
  -	*Limitations:* Although convenient, tying household mains electricity straight to an often-unsecured wireless network defies fundamental security principles. Furthermore, fail-safes are generally software-driven through the cloud rather than relying on localized hardware interrupts.

### **3.2 Heavy Machinery / Industrial Monitors**
Hardware setups like DIN-mounted PZEM monitors offer exceptionally precise, low-latency AC tracking required for industrial settings.
  -	*Limitations:* These setups entirely lack an abstraction layer for consumer interaction. There is no user-friendly graphical interface, and they provide zero behavioral incentives for the user to optimize their consumption.

## **4. Objectives**
-	**Architect an Air-Gapped Safety Node:** Develop "The Sentry," a localized ESP32 edge node capable of reading 220V mains and executing a sub-second, hardware-level power cutoff for loads above 2500W.
-	**Implement Secure Intra-Device Communication:** Utilize the router-less ESP-NOW protocol (2.4GHz) to transmit telemetry between the high-voltage Sentry node and a low-voltage display node ("The Avatar").
-	**Build a Zero-Latency Digital Environment:** Create a modern, serverless Heads-Up Dashboard (HUD) leveraging Astro, React, and TursoDB for instantaneous, scroll-free data visualization.
-	**Pioneer Emotive AI Integration:** Deploy the Google Gemini 1.5 Pro API to analyze raw telemetry and dynamically formulate relatable, emotive feedback that gamifies daily power monitoring.

---

## **5. System Architecture & Methodology**
The architecture is purposefully segregated into two primary functional bounds to ensure both electrical safety and data integrity.
| System Layer | Sub-Component | Functional Description |
|--------------|----------------|----------------------|
| **Edge Hardware (HV)** | The Sentry Node | Monitors 220V lines via a PZEM-004T v3.0 sensor. Completely isolated from the internet. Employs a 5V relay for instantaneous overcurrent protection. |
| **Edge Hardware (LV)** | The Avatar Node | An internet-facing desktop companion that receives data from the Sentry via ESP-NOW. It drives an OLED and NeoPixels to display "Emotive States." |
| **Communication Bridge** | ESP-NOW & Netlify | Data traverses a localized 2.4GHz radio frequency before the Avatar pushes JSON payloads over HTTP POST to Netlify Edge Functions. |
| **Data & Application** | TursoDB & Frontend | Serverless libSQL database guarantees reliable ingestion, whilst the Astro/React dashboard handles real-time visual rendering and Gemini API calls. |

## **6. Requirements Specifications**
### **6.1 Hardware Arsenal**
| Component	| Function within Architecture |
|-------------|------------------------------|
| **ESP32 DevKit v1 (x2)** | Dual-node processing units handling edge logic, RF, and Wi-Fi. |
| **PZEM-004T v3.0** | Precision sensor for compiling 220V AC voltage, current, and wattage. |
| **5V Relay Module** | Actuator responsible for safely breaking the circuit during overloads. |
| **1.3” OLED & WS2812B LEDs** | Peripheral visual elements rendering the Avatar's active expressions. |
| **Hi-Link HLK-PM01** | AC-to-DC step-down converter to power the high-voltage edge node. |

### **6.2 Software & Frameworks**
| Technology | Application Context |
|------------|---------------------|
| **C++ / Arduino Core** | Deployed via PlatformIO for deterministic, low-level microcontroller firmware. |
| **Netlify Edge Functions** | Scalable middle-tier API layer to process incoming telemetry POST requests. |
| **TursoDB (libSQL)** | Lightweight, serverless datastore designed for extremely fast read/write edge operations. |
| **Astro & React.js** | Composition of the interactive web HUD, styled with Tailwind CSS and TypeScript. |
| **LLM API** | Natural language engine interpreting power metrics into nuanced user alerts. |

## **7. Expected Outcomes**
-	**Validation of Air-Gapped Architecture:** Demonstration that high-voltage power can be comprehensively monitored and managed without exposing the physical relay to domestic Wi-Fi routers.
-	**Immediate Threat Mitigation:** The successful execution of localized hardware power cutoffs (under 100 milliseconds) for loads spiking over the programmed 2500W threshold.
-	**Enhanced User Retention:** Delivery of an application layer that successfully replaces traditional data spreadsheets with actionable AI insights and zero-refresh UI updates, thereby elevating user engagement.

---

## **8. Future Scopes**
### **8.1 Network & Node Expansion**
-	**Mesh Topologies (ESP-MESH):** Transitioning from a point-to-point architecture to a comprehensive domestic mesh, allowing numerous high-voltage nodes to report to a unified Avatar display hub.
-	**Bi-Directional Metering:** Upgrading system logic to handle solar-inverter feedback, enabling the monitoring of localized renewable energy generation in addition to grid consumption.

### **8.2 Intelligent Software Evolution**
-	**Acoustic Interaction:** Extending the Avatar node's hardware to include I2S audio components, facilitating two-way vocal communication with the Gemini AI model.
-	**Dynamic Grid Synchronization:** Pulling localized Time-of-Use (TOU) tariffs from grid API endpoints so the system can intelligently suggest powering down specific appliances during peak pricing.

### **8.3 Digital Ecosystem Portability**
-	**Native Mobile Integration:** Re-engineering the web dashboard into a React Native framework to support critical, immediate push notifications for emergency relay cutoffs.
-	**Carbon Tracking Platform:** Converting raw wattage data into accurate CO2 emissions based on local power-plant sourcing, offering households a gamified daily "Sustainability Score."

---

## **9. Conclusion**
**ETHER** reimagines the paradigm of domestic power monitoring by proving that consumers do not have to sacrifice structural network security to achieve high-level convenience. Through the innovative use of decoupled, air-gapped ESP32 nodes, the system inherently blocks major attack vectors while actively preventing electrical fires via zero-latency edge cutoffs. By combining this robust physical layer with an emotionally responsive AI and a state-of-the-art serverless frontend, E.T.H.E.R. transcends simple data logging—it cultivates a safer, more engaged, and environmentally conscious household.
